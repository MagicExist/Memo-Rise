# Contributing to MemoRise

## Branching — GitHub Flow
- `main` is always releasable and is branch-protected.
- Work on short-lived **feature branches, one per unit**: `feature/u0-foundation`, `feature/u1-accounts-auth`, …
- Also allowed: `fix/<slug>`, `chore/<slug>`, `docs/<slug>`.

## Commits — Conventional Commits
Format: `type(scope): summary` — e.g. `feat(auth): add signup endpoint`.
Types: `feat`, `fix`, `chore`, `docs`, `test`, `refactor`, `perf`, `build`, `ci`.
Enforced by commitlint.

### 🚫 Authorship (MANDATORY)
Commits and pull requests **must not** attribute authorship or co-authorship to any AI/assistant.
No `Co-Authored-By` AI trailers, no "generated by …" footers. Human authors only.

## Pull requests
- **One PR per unit.** Keep it focused on that unit's stories.
- CI must be green (tests + property-based tests + type-check + lint + dependency scan + SBOM).
- **Squash-merge** into `main`. Delete the branch after merge.

See `README.md` for running the backend, frontend, and local Supabase.
