# U0 Foundation — Business Rules

Cross-cutting rules U0 establishes for the whole system. Later units inherit and extend them.

## R1 — Row-Level Security (US-28, SEC-5/6)
- **R1.1** Every table holding user data MUST have RLS enabled from creation.
- **R1.2** Each table MUST define per-operation policies keyed to `auth.uid()`.
- **R1.3** No user may read or mutate another user's rows — verified at the DB, not only the app.
- **R1.4** The `service_role` key is used only for genuine system operations (e.g. the signup trigger context), never as the default path for user data.

## R2 — Profile defaults (feeds U3 gamification)
- **R2.1** A new profile starts `xp = 0`, `level = 1`, `current_streak = 0`, `last_review_date = null`.
- **R2.2** `xp` is never negative and never decreases (enforced where XP is awarded, U3).
- **R2.3** `level` is stored and only changes via the atomic `rate_card` write (U3).

## R3 — Configuration validation (fail closed, SEC-15)
- **R3.1** Required env vars MUST be present and valid at startup; a missing secret aborts boot (never starts in a half-configured state).
- **R3.2** `AI_INPUT_MAX_CHARS` (~5000) and `AI_MAX_CARDS` (~20) are enforced from config, not hardcoded at call sites (U4 uses them).
- **R3.3** `ANTHROPIC_MODEL` defaults to Claude Sonnet and is overridable by env only.

## R4 — Logging & secrets (US-27, SEC-2/3)
- **R4.1** All backend logs are structured: timestamp, request/correlation id, level, message.
- **R4.2** Secrets, tokens, and PII MUST NEVER appear in logs.
- **R4.3** A correlation id is attached per request and flows through logs.

## R5 — Error handling (SEC-15)
- **R5.1** A global error handler catches unhandled exceptions and returns a generic, safe message (no stack traces or internals to clients).
- **R5.2** On error, operations fail closed — deny access / abort; never fail open.
- **R5.3** All external calls (DB, Anthropic, I/O) have explicit error handling with resource cleanup.

## R6 — AuthN gate (feeds U1)
- **R6.1** All routes are deny-by-default; a route is accessible only if explicitly public or the request carries a valid, server-verified JWT (signature, expiry, issuer/audience).

## R7 — Version control & authorship (NFR-12)
- **R7.1** GitHub Flow: one short-lived feature branch per unit; one PR per unit; squash-merge into protected `main` behind green CI.
- **R7.2** Conventional Commits for all messages.
- **R7.3** **MANDATORY:** no commit or PR may attribute authorship or co-authorship to any AI/assistant — human authors only; no `Co-Authored-By` AI trailers or "generated by" footers.
